1· Vue-cli脚手架概念： Vue 提供一个官方命令行工具，可用于快速搭建大型单页应用。该工具为现代化的前端开发工作流提供了开箱即用的构建配置。只需几分钟即可创建并启动一个带热重载、保存时静态检查以及可用于生产环境的构建配置的项目 2· 安装步骤： # 第一步： $ npm install --global vue-cli # 第二步： $ vue init webpack my-project 输入命令后，会询问我们几个简单的选项，我们根据自己的需要进行填写就可以了。 1) Project name : 项目名称 ，如果不需要更改直接回车就可以了。注意：这里不能使用大写，所以我把名称改成了vueclitest 2) Project description: 项目描述，默认为A Vue.js project,直接回车，不用编写。 3) Author：作者 4) ? Vue build (Use arrow keys): 使用箭头键 往上往下做选择 > Runtime + Compiler: recommended for most users 运行加编译 Runtime-only: about 6KB lighter min+gzip, but templates ( or any Vue-specific HTML) are ONLY allowed in .vue files - render functions are required elsewhere 5) Install vue-router? 是否安装vue的路由插件，需要安装，所以选择Y 6) Use ESLint to lint your code? 是否用ESLint来限制你的代码错误和风格。如果你是大型团队开发，最好是进行配置。 7) setup unit tests with Karma + Mocha? 是否需要安装单元测试工具Karma+Mocha，如果不需要，所以输入n。 8) Setup e2e tests with Nightwatch? 是否安装e2e来进行用户行为模拟测试，如果不需要，所以输入n。 # 第三步： $ cd my-project 安装依赖 # 第四步： $ npm run dev 安装完成，可运行 3· 接下来我们看一下文件的目录 1) bulid 里面是一些操作文件，使用npm run * 时其实执行的就是这里的文件 2) config 配置文件，执行文件需要的配置信息 3) node_modules 文件夹，这里放的就是所有依赖的模块 4) src 存放项目的源码 5) static 存放第三方静态资源的 里面有一个文件 .gitkeep 意思是当这个目录为空，也可以把这个目录提交到github里面。（因为一般github会忽略空目录） 6) .babelrc 是babe的一些配置，因为我们目前用的代码是es6，而大多数浏览器是不支持es6语法，所以我们通过babe来将es6编译成es5。里面则是一些配置 7) .gitignore git仓库会忽略掉这些目录 8) index.html 入口文件 动态插入到这个文件 9) package.json 项目的配置文件详解 通过scripts 配置一些脚本， 在开发环境下，在命令行工具中运行npm run dev 就相当于执行 node build/dev-server.j "scripts": { "dev": "webpack-dev-server --inline --progress --config build/webpack.dev.conf.js", "start": "npm run dev", "build": "node build/build.js" }, 生产环境下 的一些依赖 版本号 "dependencies": { "vue": "^2.5.2", "vue-router": "^3.0.1" }, 编译过程中的一些依赖 "devDependencies":{ } 10) README.md 项目描述文件
